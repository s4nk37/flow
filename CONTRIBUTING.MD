# Contributing to Project

## How to Contribute

- Fork the repository.
- Create a new branch (`git checkout -b feat-branch`).
- Make your changes.
- Commit your changes (`git commit -m 'Add some feature'`).
- Push to the branch (`git push origin feature-branch`).
- Create a new Pull Request.

## How to rebase

- Fetch the origin (`git fetch origin`).
- Checkout the branch you want to rebase (`git checkout feature-branch`).
- Rebase the branch (`git rebase origin/develop`).

## Naming Conventions

- Use feat for feature branches.
- Use fix for bug fix branches.
- Use hotfix for hotfix branches.
- Use chore for maintenance branches.
- Use docs for documentation branches.
- Use style for style changes.
- Use refactor for refactoring code.
- Use `PascalCase` for class names.
- Use `camelCase` for variables and functions.
- Use `snake_case` for file names.

## Code Style

- Use App_Colors for colors.
- Use App_TextStyles for text styles.
- Use SizeConfig & Responsive for responsive design.
- Use Assets for images, icons, etc.

## Reporting Issues

- Use the issue tracker to report bugs and request features.
