# ü§ù Contributing to Flow

First off, thanks for taking the time to contribute! üéâ

The following is a set of guidelines for contributing to Flow. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## üöÄ How to Contribute

### 1. Fork & Clone
Fork the repository to your own GitHub account and then clone it to your local device.
```bash
git clone https://github.com/YOUR_USERNAME/flow.git
cd flow
```

### 2. Create a Branch
Create a new branch for your feature or fix.
```bash
git checkout -b feat/amazing-feature
# or
git checkout -b fix/annoying-bug
```

### 3. Make Changes
Implement your feature or fix. Ensure you follow the [Code Style](#-code-style) guidelines.

### 4. Verify Your Code
Before committing, run our toolkit to ensure everything is clean.
```bash
./flutter_toolkit.sh
```
*   Select **Option 1 (Analyze & Fix)** to format code and fix lint issues.
*   Select **Option 18 (Run Tests)** to ensure no regressions.

### 5. Commit & Push
Commit your changes with a descriptive message.
```bash
git commit -m "feat: add amazing feature"
git push origin feat/amazing-feature
```

### 6. Open a Pull Request
Go to the original repository and open a Pull Request (PR). Provide a clear description of your changes.

## üìè Naming Conventions

We follow strict naming conventions to keep the codebase clean.

*   **Branches:**
    *   `feat/` - New features
    *   `fix/` - Bug fixes
    *   `hotfix/` - Critical fixes for production
    *   `chore/` - Maintenance tasks
    *   `docs/` - Documentation updates
    *   `refactor/` - Code refactoring
*   **Code:**
    *   Classes: `PascalCase` (e.g., `TodoRepository`)
    *   Variables/Functions: `camelCase` (e.g., `getTodos`)
    *   Files: `snake_case` (e.g., `todo_repository.dart`)

## üé® Code Style

We enforce a strict style guide using `flutter_lints` and custom rules in `analysis_options.yaml`.

*   **Colors:** Always use `AppColors` from the theme. Do not hardcode hex values.
*   **Text:** Use `AppTextStyles` for consistent typography.
*   **Responsiveness:** Use our `Responsive` utility or `SizeConfig` for adaptive layouts.
*   **Assets:** Access images and icons via the `Assets` class generated by the toolkit.

## üêõ Reporting Issues

Found a bug? Have a feature request? Please use the [Issue Tracker](https://github.com/s4nk37/flow/issues).

*   **Bugs:** Describe the issue, steps to reproduce, and expected behavior.
*   **Features:** Explain the "why" and "what" of the proposed feature.
